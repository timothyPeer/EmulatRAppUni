--- BUGS FIXED ---

1. SrmRomLoader.cpp -- push_back aggregate initializer mismatch (C2665)
   Root cause: GuestRegionSource enum field missing from positional initializer.
   Both push_back calls in populateDescriptor() rewritten with explicit field
   order: basePA, size, type, source, description, populated, readOnly,
   includeInSnapshot, hwrpbVisible.

2. SrmRomLoader.cpp -- saveSnapshot() checksum = FNV-1a seed value
   Root cause: QDataStream destructor closed QFile before footer checksum
   block ran. file.read() returned empty QByteArray, checksum computed over
   zero bytes = 0xCBF29CE484222325 (seed).
   Fix: ds.setDevice(nullptr) to detach/flush QDataStream, close file,
   reopen read-only, readAll(), compute checksum, reopen append, write
   8-byte footer. File handle never goes out of scope mid-write.

3. GuestPhysicalRegionRegistry.cpp -- addRam() includeInSnapshot = true
   Root cause: 32GB RAM region flagged for snapshot inclusion.
   snapshotRegions() returned 4 entries including 34GB RAM -- would have
   produced a ~42GB .axpsnap file.
   Fix: addRam() sets includeInSnapshot = false. RAM opt-in deferred per
   spec §10.3 (SrmSnapshotIncludeRam flag, backlog).
