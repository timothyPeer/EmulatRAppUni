# Minimal PAL RESET → SRM Boot Checklist

 

*(Alpha AXP Emulator – EmulatR)*

 

This section defines the **minimum operational steps** required for an Alpha CPU to transition from power-on/reset into a functioning SRM console environment (`>>>`). It is intentionally concrete and implementation-oriented.

 

This checklist is divided into **must-work**, **must-exist**, and **can-stub** phases so you can progress incrementally.

 

---

 

## 1. Power-On / CPU Reset Entry

 

### 1.1 Reset Entry Conditions (Architectural Truths)

 

On reset, the Alpha CPU enters **PAL mode** with tightly defined state:

 

* Processor mode: **PAL**

* Interrupts: **disabled**

* Pipeline: **empty / flushed**

* LL/SC reservations: **cleared**

* Write buffers: **drained**

* TLBs: **invalid**

* Caches: **invalid or disabled**

 

**EmulatR requirement**

 

* `AlphaCPU::reset()` must:

 

 * Flush pipeline

 * Clear FaultDispatcher queues

 * Clear LL/SC reservations

 * Clear pending IPIs

 * Enter PAL mode explicitly

 

---

 

## 2. PAL RESET Vector Dispatch (MANDATORY)

 

### 2.1 RESET Vector Resolution

 

Alpha RESET does **not** jump to SRM directly.

 

It jumps to a **PAL entry point**:

 

```

PC = PAL_BASE + (PAL_RESET_VECTOR * 64)

```

 

**EmulatR requirement**

 

* `PalVectorResolver` must:

 

 * Resolve RESET vector

 * Redirect execution into `PalService::executeRESET()`

 

---

 

### 2.2 Minimal RESET PAL Handler Responsibilities

 

Your RESET handler does **not** need full EV6 semantics initially.

 

It must perform **only these steps**:

 

#### REQUIRED

 

* Establish a **known-good stack**

* Initialize **HWPCB** fields needed for PAL execution

* Initialize **PAL shadow state**

* Initialize exception vectors (even stubbed)

* Transition into **CSERVE-based console loop**

 

#### NOT REQUIRED (initially)

 

* Full cache init

* Full memory sizing

* Full device discovery

* Full TLB population

 

---

 

## 3. Stack & Context Bring-Up (CRITICAL)

 

### 3.1 Stack Pointer Initialization

 

SRM assumes a valid stack **before** it calls anything.

 

**EmulatR requirement**

 

* Allocate a **PAL stack region**

* Set:

 

 * `R30` (SP)

 * Any PAL shadow SP register you model

 

If this is wrong → silent failure.

 

---

 

### 3.2 HWPCB Initialization (Minimal)

 

HWPCB does *not* need full correctness initially.

 

Minimum fields:

 

* Saved PC

* Saved PS

* PAL scratch / shadow registers

* Current ASN (can be 0)

 

---

 

## 4. Transition to Console Environment (SRM Expectations)

 

SRM firmware expects **PAL to provide services**, not firmware logic.

 

### 4.1 The CSERVE Contract (THIS IS THE KEY)

 

SRM does **not** directly poke devices.

 

It calls:

 

```

CALL_PAL CSERVE

```

 

Everything flows through CSERVE.

 

---

 

## 5. Minimal CSERVE Conformance Table (MVP)

 

You do **not** need full CSERVE.

 

You need **just enough** to get `>>>`.

 

### 5.1 REQUIRED CSERVE Functions (Minimal Set)

 

| CSERVE Function | Purpose        | Required Behavior        |

| --------------- | -------------- | ------------------------ |

| GETC            | Console input  | Block or poll host input |

| PUTC            | Console output | Write char to console    |

| PUTS            | String output  | Loop PUTC                |

| RESET_TERM      | Terminal init  | Can be no-op             |

| SETMODE         | Console mode   | Can be ignored           |

| GETENV          | Read env var   | Read from SRMEnvStore    |

| SETENV          | Write env var  | Write to SRMEnvStore     |

 

Everything else may return **UNIMPLEMENTED**.

 

---

 

### 5.2 EmulatR Mapping

 

**PalService**

 

* Decodes CSERVE selector

* Dispatches to ConsoleManager

 

**ConsoleManager**

 

* Routes to `OPA0:` device

 

**ConsoleOPA0Device**

 

* Implements:

 

 * `getc()`

 * `putc()`

 * `puts()`

 

This is sufficient to print:

 

```

SRM Console Vx.x

>>>

```

 

---

 

## 6. SRM Environment Store (REQUIRED)

 

SRM expects environment variables to exist.

 

### 6.1 Minimal Environment Variables

 

You must support at least:

 

| Variable       | Purpose             |

| -------------- | ------------------- |

| `bootdef_dev`  | Default boot device |

| `boot_file`    | Boot file           |

| `boot_osflags` | Flags               |

| `auto_action`  | Boot policy         |

 

**EmulatR requirement**

 

* `SRMEnvStore` must:

 

 * Store key/value pairs

 * Return defaults if missing

 * Persist optionally (later)

 

---

 

## 7. PAL Mode Exit Rules (CRITICAL)

 

### 7.1 HW_REI Is the ONLY Exit

 

SRM expects:

 

* PAL code never “falls through”

* PAL returns via `HW_REI`

 

**EmulatR requirement**

 

* `HW_REI` must:

 

 * Restore PC

 * Restore PS

 * Restore IPL

 * Exit PAL mode

 * Resume pipeline cleanly

 

If HW_REI is wrong → *nothing works*.

 

---

 

## 8. Interrupt Model (MINIMAL)

 

Initially, interrupts may be **disabled forever**.

 

That is acceptable.

 

SRM console polling works without interrupts.

 

---

 

## 9. What You Can Stub (Safely)

 

You can reach `>>>` **without**:

 

* DMA

* Disk I/O

* Network

* Full TLB shootdown

* SMP

* Cache coherency

 

SRM prompt ≠ booting an OS.

 

---

 

## 10. Boot Progression Milestones

 

Use these checkpoints:

 

### Milestone 1

 

* RESET enters PAL

* CSERVE PUTS prints text

* You see **any output**

 

### Milestone 2

 

* GETC works

* `>>>` accepts keystrokes

 

### Milestone 3

 

* SRM variables readable

* `SHOW` commands work (even stubbed)

 

### Milestone 4

 

* Disk device stub visible

* `BOOT` command parsed (may fail)

 

---

 

## 11. Common Failure Modes (Watch These)

 

| Symptom           | Likely Cause               |

| ----------------- | -------------------------- |

| Silent hang       | Stack not initialized      |

| No output         | CSERVE PUTS/PUTC missing   |

| Immediate trap    | HW_REI incorrect           |

| Random behavior   | LL/SC not cleared on RESET |

| SRM loops forever | GETC returns garbage       |

 

---

 

## 12. Recommended Next Two Tasks (Concrete)

 

### Task A — Implement Minimal RESET + CSERVE

 

* Hardcode RESET → CSERVE loop

* Ignore everything else

 

### Task B — Lock Down Console Path

 

* PalService → ConsoleManager → OPA0

* No abstraction leaks
