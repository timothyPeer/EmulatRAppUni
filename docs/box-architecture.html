BOX Architecture

 

Silicon Box

EmulatR Subsystem

IBox

IBox class + Pipeline fetch/decode stages + iGrain resolver

Responsibilities:

•Fetches PC

•Requests PC translation (ITB)

 

Updates PC via:

•sequential fetch (PC+4)

•branch prediction (CBox)

•CALL_PAL → PAL vector PC

•Fault delivery → exception entry PC

•Pipeline flush → new PC

AlphaPipeLine

•Drives the conveyor stages (IF → DE → IS → EX → MEM → WB)

•Receives “advance PC to next fetch” signals

•Receives “flush & refetch” signals from IBox, EBox, and PAL

EBox

EBox, integer register file, integer ALUs

FBox

FBox, FP register file, FPCR, dt_sfloat/tfloat/gfloat converters

MBox

MBox, SafeMemory interface, loads/stores, TLBs, write buffer, PAL engine

CBox

C-Box IPR chains, c_data, c_shift, bus emulation layer

HWPCB

HWPCBBank, globalHWPCBController, context switching

 

Interfaces:  

AlphaCPU

Own HWPCB image of all architectural registers (PC, PS, mode bits, ASNs, stack pointers)

•Coordinate traps, PAL entry, REI, IPL changes

•Retire staged PTEs

•Provide read/write accessors to registers for boxes

•Provide interface for pipeline flush or PC override
