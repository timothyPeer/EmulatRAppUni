Alpha PipeLine

The pipeline is responsible for executing the Instruction Grain life-cycle.  All processing adjacent to it is a side-effect to support the processing of the pipeline.  The pipeline orchestrates the sequence of operations for the entire life-cycle of the Alpha Instruction Grain.

Goal.

•Pipeline is a pure lifecycle engine.

•Responsible for

oIF  - Instruction Fetch (NOOP -  except that it updates the Branch Prediction machinery)

oDE - Instruction Decode (NOOP - except that is decodes and associates the grain to the correct BOX for sideeffect implementation_)

oIS - Instruction Issue (NOOP - except that is configures dual issue candidacy on the instruction grain.)

oEX - Instruction Execute

▪Call the instruction grain's execute function

•perform work in the associated BOX previously set by the DE stage;

•Branch Prediction is calculated and resolved for WB stage

oMEM - Memory Access (NOOP - grain did all the work in the EX stage)

oWB - Writeback

▪dispatch faults and handlers

▪write back Integer & FP registers

▪Commit staged PTEs

▪advance the Program Counter (or use the calculated PC from Branch Prediction);

▪Retire

 

Rules.

•Pipeline does not commit PC; It requests a redirect via APC → CPU.

•does not do memory translation

•does not perform ALU operations

•does not do load/store side effects

•does not modify registers

•does not handle exceptions

•does not handle PAL ops

•does not touch SafeMemory

•does not advance PC (except when corrected after EXECUTION (EX) or BRANCH) (EX)

•The pipeline will track RAW and WAW register hazards.

•Stall the pipeline:

oStalls never exit the pipeline

oMust not advance the IF/DE/IS stages for 1 cycle

oInstructions already in EX, MEM, WB continue normally

oMust insert a bubble (NOP) in the next stage

oPC must not advance

oNo new instruction is fetched during the stall

oNo issue of the current DecodedInstruction into the IS Stage

 

 

Responsibilities.

•Branch Prediction  (Execution + Branch Resolution)

•Provide bitwise detection of register coherency for out of order instruction sequences; detection includes:

oRAW hazard: read still-dirty register

oWAW hazard: writing to still-dirty register

oDual-Issue Conflicts

How It Works:

 

 

Initial state (m_head = 0):

┌─────┬─────┬─────┬─────┬─────┬─────┐

│  0  │  1  │  2  │  3  │  4  │  5  │

│ IF  │ DE  │ IS  │ EX  │ MEM │ WB  │

└─────┴─────┴─────┴─────┴─────┴─────┘

  ↑ m_head

 

After advanceRing() (m_head = 1):

┌─────┬─────┬─────┬─────┬─────┬─────┐

│  0  │  1  │  2  │  3  │  4  │  5  │

│ WB  │ IF  │ DE  │ IS  │ EX  │ MEM │

└─────┴─────┴─────┴─────┴─────┴─────┘

        ↑ m_head

 

After advanceRing() (m_head = 2):

┌─────┬─────┬─────┬─────┬─────┬─────┐

│  0  │  1  │  2  │  3  │  4  │  5  │

│ MEM │ WB  │ IF  │ DE  │ IS  │ EX  │

└─────┴─────┴─────┴─────┴─────┴─────┘

              ↑ m_head

 

class AlphaPipeline {

    static constexpr int STAGE_COUNT = 6;

    PipelineSlot m_slots[STAGE_COUNT];  // Fixed array

    int m_head{0};                       // Logical "start" index

    

    PipelineSlot& slot(int logicalIndex) {

        return m_slots[(m_head + logicalIndex) % STAGE_COUNT];  // ← Ring buffer!

    }

    

    void advanceRing() {

        m_head = (m_head + 1) % STAGE_COUNT;  // ← Circular advance

    }

};
