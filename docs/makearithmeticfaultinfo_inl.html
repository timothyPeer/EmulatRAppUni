MemoryFaultInfo

makeArithmeticFaultInfo

Constructs and returns a fully populated MemoryFaultInfo structure representing an arithmetic trap (non-memory fault), capturing the faulting PC, raw instruction, arithmetic trap kind, current CPU mode, and PAL-mode state for delivery to the exception/trap handling subsystem.

 

Notes (contextual, not part of the table)

 

•This function is architecturally correct in treating arithmetic traps as:

ofaultType = NONE

ono VA/PA

ono access size

•It intentionally reuses MemoryFaultInfo as a unified fault envelope, which is the right design choice for:

oconsistent trap delivery

ologging

odeferred PAL / OS handling

•The inclusion of:

olastInstrRaw()

ogetPC_Active(cpuId)

oisInPalMode()

ogetCM_Active(cpuId)

•makes this function ideal as the single choke point for arithmetic trap metadata.

•This one belongs squarely in your faultLib boundary layer, and it’s a good example of why your key/description indexing effort is worthwhile.
