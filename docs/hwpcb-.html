ASTSR    

AST Summary Register        

Asynchronous System Trap summary

 

Consideration.  

1.ASTSR is the only stateful register for AST pending

a.ASTSR holds pending AST state

b.It is the single source of truth

c.it is updated by:

i.OS / PAL logic

ii.AST posting mechanisms

d.It is not masked

e.It is not filtered

 

2. Updates affect register: ISUM - not state

 

ISUM:

•reflects current pending interrupt state

•includes AST bits

•may be masked or unmasked depending on your design choice

•must not store independent AST state

 

When ASTSR changes:

•ISUM must observe the new AST state

•ISUM must not cache stale values

•ISUM must not be updated independently

 

Adopted implementation strategies:

 

•ISUM is computed on read

•ISUM is not stored

•MFPR_ISUM builds it dynamically from:

 

Writing the ASTSR internal processor register (Section 13.3) requests an AST for any of the

four processor modes. This operation may request an AST on a formerly inactive level and

thus cause an AST interrupt. The logic required to check for this condition is:

 

ASTSR<3:0> ← {ASTSR<3:0> AND R16<3:0>} OR R16<7:4>

IF ASTEN<0> AND ASTSR<0> AND {PS<IPL> LT 2} THEN

{initiate AST interrupt at IPL 2}

DATFX  

Data Alignment Trap

MTPR_DATFX is a PAL-only instruction that updates the DATFX internal processor register.

 

The instruction performs a pure architectural state update with no immediate side effects.

 

The new DATFX value affects the handling of future data access faults and is consulted by the fault logic when such events occur.

 

Example Usage:

 

 setKSP_Active1, 0x20000000;         // Set KSP for CPU 1

quint64 pc = getSavedPC_Active0;    // Get SavedPC for CPU 0
