PS_helpers_inl

 

Function

Returns

Description

extractIPLFromPS

quint8

Extracts the Interrupt Priority Level (IPL) field from PS

insertIPLIntoPS

quint64

Inserts a new IPL value using legacy bit placement (non-PS_IPL_MASK path)

setIPLInPS

quint64

Sets IPL field in PS using architectural PS_IPL_MASK definition

extractCMFromPS

quint8

Extracts Current Mode (CM) field (K/E/S/U) from PS

setCMInPS

quint64

Updates Current Mode (CM) field in PS

isIPLAboveLevel

bool

Checks whether current IPL exceeds a given threshold

isInterruptPendingInPS

bool

Tests whether interrupt-pending (IP) bit is set in PS

setInterruptPendingInPS

quint64

Sets or clears the interrupt-pending (IP) bit in PS

 

⚠️ Important note:

insertIPLIntoPS() uses a different IPL bit position (<20:16>) than the rest of this header (<12:8>).

This is either:

 

legacy code

an EV4/early-model artifact

 

or a bug

 

I recommend tagging it IPL-Legacy or removing it once confirmed unused.
