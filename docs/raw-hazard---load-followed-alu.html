RAW Hazard - Load Followed ALU

Load followed by dependent ALU

 

Scenario:

 

A: LDQ R1, 0(R2)

B: ADDQ R3, R1, R4 (B needs R1 written by A)

 

Goal: B must not read R1 until A has produced it and WB has committed.

 

Correct behavior with scoreboard + stall

 

We use:

 

•At IS cycle for B, check RAW hazard:

•isRegDirty(B, B.ra) || isRegDirty(B, B.rb)

 

If dirty → s1.stalled = true; for B’s slot.

 

Timeline with a one-cycle stall (simplest case; you can extend if MEM latency is modeled longer):

 

Cycle   IF    DE    IS    EX    MEM   WB

---------------------------------------------

1       A     —     —     —     —     —

2       B     A     —     —     —     —

3       —     B     A     —     —     —

4       —     —    [A]    A     —     —

              ^      ^

              |      |

           DE slot   IS slot for A issues to EX

 

5       —     —     B    [A]    —     —

                   ^     ^

                   |     |

             B reaches IS; RAW hazard check:

             R1 is DIRTY → B.stalled = true

             B stays in IS, does *not* advance

 

6       —     —     B     —     A     —

                         ^      ^

                         |      |

                        EX      MEM/WB finishing A

                      (depending on your model)

 

7       —     —     B     —     —     A

                                   ^

                                   |

                           WB: A writes R1

                           clearRegisterDirty(R1)

 

8       —     —     B     —     —     —

                   ^

                   |

           Now R1 is clean → no RAW hazard

           B can advance into EX in next cycle

 

9       —     —     —     B     —     —

10      —     —     —     —     B     —

11      —     —     —     —     —     B

 

Legend:

 

•A marks R1 dirty at IS (when it issues).

oWhen B reaches IS, isRegDirty(B, R1) returns true → B.stalled = true.

oB sits in the IS slot until WB of A has cleared the dirty bit.

•Only after that does B issue and read the correct R1 value.
