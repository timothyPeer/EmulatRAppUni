IPR_IC_FLUSH_inl

 

Returned

Function Name

Inferred Description

void

writeIPR_ICTL

Handles writes to the I_CTL IPR: masks read-only fields, preserves architectural bits, updates control state, flushes the instruction cache model, clears speculative prefetch state, and enforces I-box side effects required by Alpha IPR semantics.

 

Pure IPR logic — no CPU side effects

applyIctlWriteSemantics

Masks illegal writes to I_CTL, preserves read-only fields, and returns the architecturally correct new raw I_CTL value.

 

I-BOX / fetch-side effects

invalidateInstructionFrontend

Notes (contextual, not part of the table)

•This function implements architectural write semantics for the I_CTL register:

oPreserves RO fields: CHIP_ID, BIST_FAIL, SL_RCV

oMasks illegal writes before committing raw

•Performs mandatory side effects of an I_CTL write:

oInstruction cache invalidation

oPrefetch / speculation reset

•Correctly lives on a hot path (AXP_HOT) because it affects fetch and decode behavior.

•This function conceptually bridges:

oIPR semantics

oI-box microarchitectural state

oEmulator-side speculative machinery
