QtRandomCompat

Determinism Note:

 

Uses host-provided randomness (Qt RNG or qrand). Results are not replay-safe unless externally seeded or overridden. Any subsystem consuming this API is implicitly non-deterministic.

 

Function/Macro

Name

Description

Consuming Subsystems

Q_EMULATR_RANDOM_INT(N)

Generate bounded random integer

Version-safe macro that returns a uniformly distributed integer in [0, N), using QRandomGenerator for Qt ≥5.10 and qrand() fallback otherwise

Scheduler, replacement policy (TLB/cache), fuzz/testing paths, non-deterministic heuristics

seedEmulatrRandom()

Initialize legacy random seed

Seeds qrand() once for Qt <5.10; no-op for modern Qt where QRandomGenerator is self-seeded

Application startup, legacy build support

 

Notes (design-level, not part of the table)

•This header is infrastructure-only and correctly avoids any Alpha-specific coupling.

•The macro form is appropriate here because:

oIt allows use in constexpr-unfriendly contexts.

oIt avoids introducing an ABI-visible helper symbol.

•Good separation: policy (who uses randomness) stays elsewhere; mechanism (how randomness is produced) lives here.
