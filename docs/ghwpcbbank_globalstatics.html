Host the Alpha Hardware Process Context Block (HWPCB )

 

#ifndef HWPCB_SLOT_COUNT

#define HWPCB_SLOT_COUNT 8  // Typical: 0=Active, 1..7=PALtemp, or more for multiprocess

#endif

 

Considerations: Each CPU requires two slots:  (1) Active context, (2) SWPCTX "saved" context

 

Notes:

•gHWPCBBank is global and indexable by CPU ID and HWPCB slot.

•context_save and context_restore use plain copy assignment (so ensure HWPCB is POD or implements copy/clear).

•resetAll calls your HWPCB's reset() on every slot.

•Scales to expand the slot count for more PALtemp, process, or interrupt context slots as needed.

•The CPU always accesses the global bank via [cpuId][slot].

 

During SWPCTX or when context state must be save:

 

quint8 cpuid = 0; 

 

// Save active CPU 0’s context into shadow slot 5

hwpcb_context_save(cpuid);     // slot 0 -> slot 5

 

// Restore from shadow slot 5 back into slot 0 (active)

hwpcb_context_restore(cpuid);  // slot 5 -> slot 0

 

Reset Operations:

•IPRStorage::reset()

•gHPWCBank()::resetAll()

 

If you want multiple context save slots, you can increase the offset (e.g. for nested interrupts).

--------------------------------------------------------------------------------------------------------

// Usage Example:

//

// Initialization (at startup)

 gHWPCBBank->resize(numCPUs, HWPCB_SLOT_COUNT);   e.g., 4 CPUs, 8 slots each

 

// Access current/active slot (e.g., slot 0)

 HWPCB& active = gHWPCBBank->operator()(cpuId, 0);

 

//  Save/restore PALtemp slots

 gHWPCBBank->context_save(cpuId, /*src=*/0, /*dst=*/1);       Save active to PALtemp[1]

 gHWPCBBank->context_restore(cpuId, /*src=*/1, /*dst=*/0);    Restore PALtemp[1] to active

 

//  Reset all slots for all CPUs

 gHWPCBBank->resetAll();
