Export Symbols

===============================================================================

              C++/Qt Symbol Export Cheat Sheet for Shared Libraries

===============================================================================

 

Overview:

---------

When building a shared library (DLL/SO), you must mark exported symbols so they

are visible to code that links against your library. This is usually done via

an EXPORT macro and a global header (e.g., corelib_global.h).

 

-------------------------------------------------------------------------------

1. Defining the EXPORT Macro

-------------------------------------------------------------------------------

- Create a header (e.g., corelib_global.h):

 

   #ifndef CORELIB_GLOBAL_H

   #define CORELIB_GLOBAL_H

   #include <QtCore/qglobal.h>

   #if defined(CORELIB_LIBRARY)

     #define CORELIB_EXPORT Q_DECL_EXPORT

   #else

     #define CORELIB_EXPORT Q_DECL_IMPORT

   #endif

   #endif // CORELIB_GLOBAL_H

 

- When building the library itself, define the macro (e.g., CORELIB_LIBRARY).

- When linking _to_ the library, do NOT define it.

 

-------------------------------------------------------------------------------

2. Exporting Classes, Structs, Enum Classes

-------------------------------------------------------------------------------

 

- To export a class:

   class CORELIB_EXPORT MyClass { ... };

 

- To export a struct:

   struct CORELIB_EXPORT MyStruct { ... };

 

- To export an enum class:

   enum class CORELIB_EXPORT MyEnumClass { ... };

 

   (Note: Enum classes rarely need exporting unless used as a symbol across DLLs)

 

-------------------------------------------------------------------------------

3. Exporting Global Variables and Constants

-------------------------------------------------------------------------------

 

- For `static constexpr` (in header): No EXPORT needed.

   static constexpr int MyValue = 42;   // Each TU gets its own copy.

 

- For a shared (non-static) constant:

   // In header

   extern CORELIB_EXPORT const int MyExportedValue;

   // In source (.cpp)

   CORELIB_EXPORT const int MyExportedValue = 123;

 

-------------------------------------------------------------------------------

4. General Guidelines and Pitfalls

-------------------------------------------------------------------------------

- The EXPORT macro must appear *immediately before* the class/struct/enum

 keyword in the declaration.

- Only decorate *declarations*, not method implementations or member functions

 inside the .cpp file.

- EXPORT macro not needed for inline methods inside classes marked for export.

- Avoid exporting private/internal types. Export only API-facing classes/types.

 

-------------------------------------------------------------------------------

5. Platform-Specific Notes

-------------------------------------------------------------------------------

- Windows (MSVC): __declspec(dllexport) / __declspec(dllimport)

- Linux/Mac (GCC/Clang): __attribute__((visibility("default"))) / usually not needed

- Qt’s Q_DECL_EXPORT and Q_DECL_IMPORT handle these differences for you.

 

-------------------------------------------------------------------------------

6. Auto-Exporting All Symbols (Not Recommended)

-------------------------------------------------------------------------------

- GCC/Clang: Compile with -fvisibility=default (not recommended for API hygiene).

- MSVC: Use linker /EXPORT or .DEF files (rarely used in Qt projects).

- **Best Practice**: Explicitly decorate only what should be exported.

 

-------------------------------------------------------------------------------

7. Precompiled Headers (PCH)

-------------------------------------------------------------------------------

- Always #include the global export macro header *after* PCH includes.

 

-------------------------------------------------------------------------------

8. Example Summary

-------------------------------------------------------------------------------

   // corelib_global.h

   #include <QtCore/qglobal.h>

   #if defined(CORELIB_LIBRARY)

     #define CORELIB_EXPORT Q_DECL_EXPORT

   #else

     #define CORELIB_EXPORT Q_DECL_IMPORT

   #endif

 

   // myclass.h

   #include "corelib_global.h"

   class CORELIB_EXPORT MyClass { ... };

 

   // mystruct.h

   struct CORELIB_EXPORT MyStruct { ... };

 

   // myenum.h

   enum class CORELIB_EXPORT MyEnumClass { ... };

 

-------------------------------------------------------------------------------
