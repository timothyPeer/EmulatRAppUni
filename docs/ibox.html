IBox

IBox performs decode + issue, NOT execution

 

Dispatches instruction grain units to the associated BOX; does not “schedule” globally.

 

Branch Prediction.

•Primary role - decides which to PC to fetch next.

•IBox does not train anything; It simply uses existing predictor state to fetch ahead.

•IBox must NOT update PC architecturally — only supply predicted PC as “speculative next fetch PC”.

 

Consumes.

•Branch Prediction (Fetch + Front-end Prediction Consumer)

 

Interfaces.

 

predictBranch_IBox()

getPredictedBranchTarget_IBox()

flush()

 

Responsibilities.

 

| Task                           | Function                                                 |

| ------------------------------ | -------------------------------------------------------- |

| Fetch next instruction         | `fetchNext()`                                            |

| Decode                         | `decodeInstruction()`                                    |

| Use predictor                  | calls CBox prediction interface                          |

| Supply predictions to pipeline | returns `FetchResult{ predictedTaken, predictedTarget }` |

| Never resolves branches        | Only pipeline EX resolves and trains                     |
