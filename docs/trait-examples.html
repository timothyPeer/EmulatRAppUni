Alpha PTE Traits Usage Guide

Overview

--------

The Alpha PTE subsystem uses *traits* to adapt the canonical AlphaPTE format to the

specific page table layouts and access patterns of each Alpha CPU generation (EV4, EV5, EV6).

Traits provide static helpers for decoding, encoding, and extracting VA and permission info

from hardware-format PTEs and tags.

 

This guide explains:

•How to select and use PTETraits specializations

•How to convert between raw IPR (hardware) PTEs and canonical AlphaPTEs

•How to use these traits with the SPAMShardManager (PTE cache/TLB manager)

•How to connect these in your CPUStateIPRInterface and page table handling code

 

Key Types

---------

•AlphaPTE         : Canonical struct for all Alpha PTEs (see alpha_pte_core.h)

•PTETraits        : Template providing static decode/encode methods for a CPU gen/layout

•SPAMShardManager : TLB/PTE cache template (uses traits for all encoding/decoding)

•CPUStateIPRInterface : Your CPU/SoC context interface for IPR and TLB operations

 

Selecting the Right Traits

--------------------------

Choose the appropriate traits specialization for your target. Example for EV6 ITB and DTB:

 

    // EV6 ITB traits (Instruction TLB)

    using ItbTraits = PTETraits<CPUFamily::EV6, 44U>; // 44-bit VA

 

    // EV6 DTB traits (Data TLB)

    using DtbTraits = Ev6_DtbPteTraits<CPUFamily::EV6, 44U>;

 

See: alpha_pte_traits_ev6_itb.h and alpha_pte_traits_ev6_dtb.h

 

Example: Decoding a PTE from a raw ITB_PTE register value

---------------------------------------------------------

Suppose you have just written to the ITB_PTE register (e.g., via MTPR). You want to canonicalize this hardware-format PTE using the trait:

 

    quint64 rawItbPte = /* value from IPR write */;

   AlphaPTE canonicalPte = ItbTraits::fromItbPteWrite(rawItbPte);

 

Example: Decoding VA from an ITB_TAG register

---------------------------------------------

    quint64 rawItbTag = /* value from IPR ITB_TAG */;

    quint64 va = ItbTraits::decodeVAFromTag(rawItbTag);

 

Example: Inserting a PTE into the TLB using Traits and SPAMShardManager

----------------------------------------------------------------------

This is the core of your stageITBPTE helper

 

    // 1. Canonicalize hardware-format PTE

    AlphaPTE canonicalPte = ItbTraits::fromItbPteWrite(rawPteWriteFormat);

 

    // 2. Recover VA from staged ITB_TAG

    quint64 va = ItbTraits::decodeVAFromTag(stagedItbTag);

 

    // 3. Obtain current ASN from the CPU or IPR

    quint8 asn = getCurrentASN();

 

    // 4. Determine global mapping (ASM == 0 is global)

    bool isGlobal = !canonicalPte.getAsm();

 

    // 5. Insert into TLB/SPAM

    itbManager->tlbInsert(cpuId, Realm::I, 0, va, asn, isGlobal, canonicalPte);

 

Example: Full DTB Bank Handling Using Traits (stageDTB0PTE)

-----------------------------------------------------------

    // 1. Canonicalize hardware-format PTE

    AlphaPTE canonicalPte = DtbTraits::fromDtbPteWrite(rawDtbPte);

 

    // 2. Recover VA from staged DTB_TAG0

    quint64 va = DtbTraits::decodeVAFromTag(stagedDtbTag0);

 

    // 3. Get ASN

    quint8 asn = getCurrentASN();

 

    // 4. Determine global mapping

    bool isGlobal = !canonicalPte.getAsm();

 

    // 5. Insert into D-stream TLB/SPAM

    dtbManager->tlbInsert(cpuId, Realm::D, 0, va, asn, isGlobal, canonicalPte);

 

Working with PTEView and Trait-Based Abstraction

------------------------------------------------

If you want a view object that wraps a canonical PTE and provides trait-based access:

 

    PTEView<ItbTraits> pteView(canonicalPte);

    // Access validity, PFN, permission, etc.:

    bool valid = pteView.isValid();

    quint32 pfn = pteView.pfn();

    PermMask pmask = pteView.permMask();

 

See: alpha_pte_view.h

 

Usage Pattern in CPUStateIPRInterface

-------------------------------------

•Use traits to canonicalize or decode hardware-format PTEs and tags (see stageITBPTE, stageDTB0PTE).

•Use SPAMShardManager's tlbInsert, tlbLookup, tlbTBIS, and invalidateASN to manage the TLB/cache.

 

Example: Integrating Traits with TLB Insert (from your interface)

-----------------------------------------------------------------

    // Insert ITB entry (instruction translation)

    bool CPUStateIPRInterface::insertITB(quint64 va, quint8 asn, const AlphaPTE& pte, bool global) noexcept {

        return itbManager->tlbInsert(cpuId, Realm::I, 0, va, asn, global, pte);

    }

 

    // Lookup DTB entry (data translation)

    bool CPUStateIPRInterface::lookupDTB(quint64 va, quint8 asn, quint32& outPfn, PermMask& outPermMask) const noexcept {

        return dtbManager->tlbLookup(cpuId, Realm::D, 0, va, asn, outPfn, outPermMask);

    }

 

Advanced Example: Specializing for EV6/EV5/EV4 at Compile Time

--------------------------------------------------------------

    #if TARGET_CPU == CPUFamily::EV6

        using ItbTraits = PTETraits<CPUFamily::EV6, 44U>;

        using DtbTraits = Ev6_DtbPteTraits<CPUFamily::EV6, 44U>;

    #elif TARGET_CPU == CPUFamily::EV5

        using ItbTraits = PTETraits<CPUFamily::EV5, 43U>;

        using DtbTraits = PTETraits<CPUFamily::EV5, 43U>;

    #else

        using ItbTraits = PTETraits<CPUFamily::EV4, 32U>;

        using DtbTraits = PTETraits<CPUFamily::EV4, 32U>;

    #endif

 

Now all your decode/encode logic is handled through the chosen traits type.

 

Trait-Driven TLB Insert (SPAMShardManager::tlbInsert)

-----------------------------------------------------

    bool tlbInsert(quint16 cpuId,

                   Realm realm,

                   quint8 sizeClass,

                   quint64 va,

                   quint8 asn,

                   bool isGlobal,

                   const AlphaPTE& pteRaw) noexcept

    {

        using T = Traits;

        // Prepare tag

        const auto tag = makeTag(va, realm, sizeClassFor(sizeClass));

        Entry e{};

        e.tag = tag;

        e.asn = asn;

        e.flags.global = isGlobal;

        e.flags.transitioning = false;

        e.asnGenAtFill = m_asnEpochs[cpuId][asn].loadRelaxed();

        e.pfn = static_cast<quint32>(T::pfn(pteRaw));

        e.permMask = T::permMask(pteRaw);

        e.pteRaw = pteRaw;

        auto& buck = bucket(cpuId, realm, sizeClass, bucketIndex(tag));

        return buck.insert(e);

    }

Best Practices

--------------

•Always decode hardware-format PTEs and tags using the correct traits.

•Never assume a specific layout—always use traits helpers for cross-generation support.

•Use PTEView when you need safe, readable field access for generic code.

•Use SPAMShardManager for all translation buffer (TLB/PTE cache) management.
