Dual Issue Hazard

Case Dual Issue:

 

Slot(2): A

Slot(3): B

 

Case: A and B both write the same register (WAW across pair)

 

A: ADDQ R1, R2, R3

B: SUBQ R4, R5, R1 (same destination R1)

 

Dual-issue logic must detect:

 

// Second instruction must not conflict with first

if (di1.rc == di2.rc && di1.rc != 31) {

    s2.stalled = true;

    return;

}

 

 

Simplified timing (only showing IS and EX):

 

Cycle   IF   DE    IS            EX

------------------------------------------

1       A    —     —             —

2       B    A     —             —

3       C    B     [A,B]         —

                ^   ^

                |   |

          s1 = A, s2 = B in IS

 

          Hazard check:

          di1.rc == di2.rc == R1 → WAW pair hazard

          → s2.stalled = true; do not issue B

 

4       D    C     B (stalled)   A

                    ^

                    |

                 B did not issue

                 A went to EX

 

5       E    D     B             —

6       —    E     B             A (WB)

7       —    —     B             —

8       —    —     —             B

 

 

Here, the pair hazard logic ensures:

•A issues and writes R1

•B is held in the IS slot until R1 is clean

•Only then can B issue — either alone or paired with some later instruction
