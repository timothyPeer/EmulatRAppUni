### Chapter Structure Observations

 

**Chapters 1–11** are *behavioral / architectural* — they define what the emulator guarantees.

**Chapters 12–15** are *implementation-facing* — they describe how code delivers those guarantees.

**Chapters 16–22** continue the implementation-facing pattern, each drilling into a specific subsystem.

 

The naming convention shifted at Chapter 12:

- Chapters 1–11 use **en-dash** in titles: `Chapter 1 - System Overview`

- Chapters 12–22 use **em-dash**: `Chapter 12 – AlphaCPU Core`

 

This is cosmetic and does not affect DevSpace rendering, but should be consistent going forward. Recommendation: use em-dash (`–`) for all subsequent chapters.

## 3. Cross-Reference Format Analysis

 

### 3.1 Template Format (Canonical — from user-provided XML)

 

The template shows See Also references using `<link>` tags:

 

```xml

<para styleclass="SeeAlso">See Also: <link displaytype="text" defaultstyle="true"

 type="topiclink" href="Chapter-4_3-Relationship-Betwe">4.3 Execution Flow:

 Grains, Pipeline, and Boxes</link>; <link displaytype="text" defaultstyle="true"

 type="topiclink" href="Chapter-4_10-Cross-Box-Interac">4.10 Cross-Box

 Interaction Rules</link>.</para>

```

 

**Link attributes:**

- `displaytype="text"` — always text

- `defaultstyle="true"` — use default link style

- `type="topiclink"` — internal topic cross-reference

- `href="..."` — truncated topic slug (DevSpace auto-generates from topic title)

 

### 3.2 Chapters 1–13 (Pre-Session)

 

In the docx export, these See Also links render as **hyperlinked text** using internal anchors. The docx relationship IDs map to anchors like:

 

```

#Chapter_4_3_Relationship_Betwe4D189221

#Chapter_5_11_Load_Locked___StoB2134075

#Chapter_14___Execution_DomainsFCC67370

```

 

The pattern: `#` + title words with underscores + 8-char hex hash. This is auto-generated by Help&amp;Manual during export. The XML source uses the shorter `href` slugs shown in section 3.1.

 

**Reference style in chapters 1–13:** Primarily chapter-level and section-level references to other documentation chapters, with occasional source file references.

 

### 3.3 Chapters 14–15 (This Session)

 

These chapters use **plain text** in the See Also para — no `<link>` tags:

 

```xml

<para styleclass="SeeAlso">See Also: Chapter 5 – Memory System Architecture

 (behavioral contract); Chapter 6 – Serialization and Stall Model; ...</para>

```

 

**Additionally**, chapters 14–15 introduce a **source file reference pattern** not present in chapters 1–13:

 

```xml

<para styleclass="SeeAlso">See Also: memoryLib/GuestMemory.h (~286 lines) – PA

 router class; memoryLib/SafeMemory.h (~253 lines) – Offset-based RAM

 interface; ...</para>

```

 

This is a valid and valuable addition (implementation-facing chapters benefit from direct file references), but should be separated from chapter/section cross-references for clarity.

 

### 3.4 Recommendation for Chapters 16–22

 

**Use `<link>` tags for chapter/section references.** This is the canonical format from the template and enables DevSpace auto-linking.

 

**Use plain text for source file references.** Source files are not topics and cannot be linked.

 

**Proposed format for subsequent chapters:**

 

```xml

<para styleclass="SeeAlso">See Also: <link displaytype="text" defaultstyle="true"

 type="topiclink" href="Chapter-5-Memory-System-Archit">Chapter 5 – Memory System

 Architecture</link>; <link displaytype="text" defaultstyle="true"

 type="topiclink" href="15_3-GuestMemory-Region-Suppor">15.3 GuestMemory Region

 Support and PA Routing</link>; memoryLib/GuestMemory.h – PA router class.</para>

```

 

**href slug generation rule:**

Take the topic `<title>` text, replace spaces and special characters with hyphens, truncate to approximately 30 characters at a word boundary.

 

| Topic Title | href slug |

|---|---|

| `Chapter 5 – Memory System Architecture` | `Chapter-5-Memory-System-Archit` |

| `15.3 GuestMemory Region Support and PA Routing` | `15_3-GuestMemory-Region-Suppor` |

| `4.10 Cross-Box Interaction Rules` | `Chapter-4_10-Cross-Box-Interac` |

 

**Note:** When the XML is imported into Help&amp;Manual/DevSpace, the tool will auto-assign topic IDs. The `href` values can be adjusted at import time if they don't match. The important thing is to have the `<link>` structure in place so the cross-reference intent is clear.

### Formatting Rules (Confirmed from existing chapters)

 

| Element | Style/Tag | Notes |

|---------|-----------|-------|

| Chapter intro | First `<topic>` in file | No section number in title |

| Section topic | Separate `<topic>` per N.M section | `<title>` = `N.M Title` |

| Section heading | `<para styleclass="Heading1">` in header | Matches `<title>` |

| Subsection heading | `<para styleclass="Heading2">` | N.M.K numbering |

| Subsection separator | `<line style="height:1px; color:#000000;" />` | Between subsections within topic |

| Body text | `<para styleclass="Normal">` | — |

| Code/monospace inline | `<text styleclass="Code Example">` | Within `<para styleclass="Normal">` |

| Code block | `<para styleclass="Code Example">` | One para per line |

| Bold inline | `<text style="font-weight:bold;">` | — |

| See Also | `<para styleclass="SeeAlso">` | Standalone paragraph (not inline text style) |

| See Also links | `<link type="topiclink" href="...">` | For chapter/section refs |

| See Also files | Plain text | For source file refs |

| See Also separator | Semicolons | Between items |

## 7. Key Adjustments for Subsequent Chapters

 

### 7.1 Use `<link>` tags in See Also

Chapters 14–15 used plain text references. Subsequent chapters should use the `<link>` format from the template for all chapter/section cross-references. Source file references remain plain text.

 

### 7.2 Consistent em-dash in chapter titles

Use `–` (en-dash/em-dash) consistently: `Chapter 16 – Device Model & DMA`

 

### 7.3 Match forward reference targets

When writing chapters 17 and 18, ensure the topic `<title>` values create slugs that match the forward references already placed in chapters 14–15. The references use these exact chapter title strings:

- "Chapter 17 – Address Translation, TLB, and PTE"

- "Chapter 18 – Fault Dispatcher & Precise Exceptions"

 

### 7.4 Each numbered section as separate topic

Confirmed working pattern from chapters 14–15. Each `N.M` section gets its own `<topic>` block. Chapter intro is the first topic (no section number).

 

### 7.5 Source file references in See Also

Implementation chapters (14+) appropriately include source file references. Continue this pattern:

`sourceDir/FileName.h (~NNN lines) – Brief description`

 

### 7.6 Cross-reference density target

Chapters 1–13 average **12.4 See Also entries per chapter**. Chapters 14–15 average **12 See Also entries per chapter**. Maintain similar density for chapters 16–22.

 

### 7.7 Do NOT include "Purpose of This Chapter" section

Chapters 14–15 omit the standalone "Purpose" section found in chapters 1–13. For the implementation-facing chapters 16–22, the chapter intro topic serves this purpose. This is a deliberate structural shift starting at chapter 14.
