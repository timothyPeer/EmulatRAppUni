globalIRQController)

Interrupt Controller (IPL arbitrator and delivery).

 

Considerations:

•Thread Safe.

•Maintains a vector for the number of CPUs that are registered-attached to it.

•The gIRQController() must be initialize with the CPU count (constructor requires it):

 

// At system init (before SMPManager/CPUs start):

 

globalIRQController().init(numCpus);

 

Special Note:

// The maximum number of CPUs are fixed at compile time.  This is a convenience extension only.

 

#define MAX_CPUS 64

 

 

Accessors:

 

 

Best Practice:

•Global, indexed access: Works for all SMP and per-CPU IRQ control.

•Singleton lifetime: System-wide, resettable at runtime.

•No need for per-CPU controller objects—one global, multi-CPU-aware instance is enough.

•Externally accessible from anywhere (PAL, DeviceManager, CPU, MMIO, etc).
