Architectural Notes (concise, high-value)

•Pure storage + bitfield accessors

•No AlphaCPU dependency — clean separation of state vs behavior.

•EV6-accurate I_CTL modeling

•Bit layout matches documented IBox control register fields.

•Flush decoupling is correct

•ic_flush is modeled as write-only trigger; execution belongs elsewhere.

•Branch prediction & VA format live here

•Correctly places BP mode and VA width control in the IBox domain.

 

This header is exactly what you want for:

•PALcode reads/writes

•HW_MFPR / HW_MTPR

•Snapshot / restore paths

•SMP-safe per-CPU IPR state
