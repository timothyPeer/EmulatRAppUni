Cache Replacement and Invalidation Policies

Overview

--------

The cache replacement and invalidation policies are critical components of the PTE subsystem's cache management strategy. These policies determine how cache entries are selected for eviction when the cache is full and how invalidation operations are performed.

 

The key policy classes are:

1. ReplacementPolicyBase: The base class for all victim selection policies.

2. SRRIPPolicy: Static Re-Reference Interval Prediction policy for victim selection.

3. ClockPolicy: Clock-based replacement policy for victim selection.

4. RandomPolicy: Random replacement policy for victim selection.

5. DefaultInvalidationStrategy: The default invalidation strategy.

 

ReplacementPolicyBase

---------------------

ReplacementPolicyBase is an abstract base class that defines the interface for all victim selection policies. It provides two pure virtual methods that must be implemented by derived policy classes:

 

    virtual unsigned selectVictim(const Entry* entries) noexcept = 0;

    virtual void recordAccess(const Entry& entry, unsigned way) noexcept = 0;

 

•selectVictim: Selects a victim entry for eviction when the cache is full. Takes an array of cache entries and returns the index of the selected victim.

•recordAccess: Called when a cache entry is accessed. Allows the policy to update its internal state based on the access pattern.

 

Derived policy classes must implement these methods to define their specific victim selection strategy.

 

SRRIPPolicy

-----------

SRRIPPolicy is a victim selection policy based on the Static Re-Reference Interval Prediction (SRRIP) algorithm. It assigns a re-reference prediction value (RRPV) to each cache entry and selects the entry with the highest RRPV as the victim.

 

Key features of SRRIPPolicy:

•Maintains a 2-bit RRPV for each cache entry.

•On a cache hit, the RRPV of the accessed entry is set to 0 (most recently used).

•On a cache miss, the victim is selected as the entry with the highest RRPV.

•If multiple entries have the same highest RRPV, the first one encountered is selected as the victim.

 

SRRIPPolicy provides a balance between recency and frequency of access, making it effective for workloads with temporal locality.
