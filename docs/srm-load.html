│ PAL_BASE REGISTER (IPR 0x140)                                    │

├──────────────────────────────────────────────────────────────────┤

│                                                                  │

│ Purpose:  Points to PALcode base address in memory              │

│ At Reset: 0x0000000000000000 (per EV6 HRM section 4.1.1)       │

│ Set By:   SROM bootstrap code via MTPR instruction              │

│ Used By:  CPU to calculate CALL_PAL vector addresses            │

│                                                                  │

│ Formula:  PAL_VECTOR = PAL_BASE + (function * 64)               │

│ Example:  CALL_PAL #0x03 → Jump to (PAL_BASE + 0xC0)           │

│                                                                  │

└──────────────────────────────────────────────────────────────────┘

 

┌──────────────────────────────────────────────────────────────────┐

│ REAL HARDWARE BOOT (5 Stages)                                    │

├──────────────────────────────────────────────────────────────────┤

│                                                                  │

│ 1️⃣  RESET                                                        │

│     PC = RESET_VECTOR (e.g., 0x1FC00000)                        │

│     PAL_BASE = 0                                                 │

│     CPU executes from ROM/flash                                  │

│                                                                  │

│ 2️⃣  SROM (Serial ROM Bootstrap)                                  │

│     - Initialize memory controller                               │

│     - Load PALcode → 0x00000000                                 │

│     - Load SRM → 0x20000000                                     │

│     - MTPR PAL_BASE, 0x00000000  ← SET HERE!                    │

│     - Jump to PALcode entry                                      │

│                                                                  │

│ 3️⃣  PALcode Initialization                                       │

│     - Initialize IPRs, TLB, caches                              │

│     - Setup interrupt vectors                                    │

│     - Jump to SRM                                                │

│                                                                  │

│ 4️⃣  SRM Firmware                                                 │

│     - Display console "P00>>> "                                 │

│     - Accept commands                                            │

│     - When "boot" typed, load OS                                │

│                                                                  │

│ 5️⃣  Operating System                                             │

│     - OS takes control                                           │

│     - Uses CALL_PAL for system services                         │

│                                                                  │

└──────────────────────────────────────────────────────────────────┘

 

┌──────────────────────────────────────────────────────────────────┐

│ YOUR EMULATOR (C++ PAL Emulation)                                │

├──────────────────────────────────────────────────────────────────┤

│                                                                  │

│ Initialization:                                                  │

│   ✅ Load clipper.bin → memory[0x20000000]                       │

│   ✅ Set PAL_BASE = 0 (per HRM, not used)                        │

│   ✅ Set PC = 0x20000000                                         │

│   ✅ Set PAL mode = true                                         │

│   ✅ Set owner = SRM                                             │

│   ✅ Start CPU execution                                         │

│                                                                  │

│ Execution:                                                       │

│   ✅ SRM code executes from clipper.bin                          │

│   ✅ When CALL_PAL decoded, trap to C++                          │

│   ✅ PalService::executeCSERVE() runs (not Alpha code!)          │

│   ✅ ConsoleManager performs I/O                                 │

│   ✅ Return to SRM code                                          │

│                                                                  │

│ PAL_BASE Usage:                                                  │

│   ❌ NOT used (CALL_PAL traps to C++, not Alpha PAL)             │

│   ❌ NOT consulted for vector calculation                        │

│   ✅ Set to 0 for HRM compliance only                            │

│ Real Hardware:                                                   │

│   SRM: CALL_PAL #0x03                                           │

│   CPU: Calculate PAL_BASE + (0x03 * 64) = 0x000000C0           │

│   CPU: Jump to 0x000000C0                                       │

│   PAL: Execute Alpha code at 0x000000C0                         │

│                                                                  │

│ Your Emulator:                                                   │

│   SRM: CALL_PAL #0x03                                           │

│   CPU: Decode opcode 0x00, function 0x03                        │

│   CPU: Trap to C++ handler                                      │

│   C++: PalService::executeCSERVE()                              │

│   C++: Examine R16 for function code                            │

│   C++: ConsoleManager performs I/O                              │

│   C++: Return to SRM code                                       │

│   (PAL_BASE never consulted!)                                   │

│                                                                  │

│ INITIALIZATION CODE TEMPLATE                                     │

│                                                                  │

│ // 1. Load SRM firmware                                         │

│ memcpy(m_memory + 0x20000000, clipper_data, clipper_size);     │

│                                                                  │

│ // 2. Set PAL_BASE (HRM compliance, not used)                   │

│ cpu->writeIPR(0x140, 0x0000000000000000);                       │

│                                                                  │

│ // 3. Set PC to SRM entry                                       │

│ cpu->setPC(0x20000000);                                         │

│                                                                  │

│ // 4. Set PAL mode                                              │

│ cpu->setPALMode(true);                                          │

│                                                                  │

│ // 5. Set owner                                                 │

│ cpu->setOwner(CPUOwner::SRM);                                   │

│                                                                  │

│ // 6. Park secondary CPUs                                       │

│ for (i = 1; i < numCPUs; i++)                                   │

│     cpus[i]->setOwner(CPUOwner::Parked);                        │

│                                                                  │

│ // 7. Start execution                                           │

│ cpu->run();  // Executes clipper.bin from 0x20000000           │

┌──────────────────────────────────────────────────────────────────┐

│                                                                  │

│ Question: "What is the order of operations to set PAL_BASE?"    │

│                                                                  │

│ Answer:                                                          │

│   Real Hardware: SROM sets it after loading PALcode             │

│   Your Emulator: Set to 0 during init (not used)                │

│                                                                  │

│ Question: "Does PAL_BASE store firmware load address?"          │

│                                                                  │

│ Answer:                                                          │

│   No - PAL_BASE points to PALcode, not SRM firmware             │

│   SRM loads at 0x20000000 (separate from PAL)                   │

│   In your emulator, PAL_BASE is irrelevant (C++ emulation)      │

│                                                                  │

│ Key Insight:                                                     │

│   Your emulator uses C++ PAL handlers, not Alpha PALcode        │

│   Therefore PAL_BASE is NOT USED                                │

│   Set it to 0 for HRM compliance only                           │

│                                                                  │

└──────────────────────────────────────────────────────────────────┘
