IRQ Routing Requirements

IRQ-REQ-001:

Provide per-vector routing policy (FIXED_CPU, ROUND_ROBIN, LOWEST_IPL, BROADCAST).

IRQ-REQ-002:

Store per-vector hose/device metadata for SCSI/NIC/IO routes.

IRQ-REQ-003:

Deliver interrupts to target CPU using the correct policy.

 

Pending Interrupt Tracking

IRQ-REQ-010:

Track pending interrupts using an atomic bitmask (pendingIPLMask).

IRQ-REQ-011:

Track per-IPL FIFO queue of posted vectors.

IRQ-REQ-012:

Provide lock-free fast-path read (hasPendingInterrupt()).

 

Interrupt Dispatch Requirements

IRQ-REQ-020:

getNextInterrupt() must: Acquire lock, Pop FIFO vector, Clear bit if queue empty, Return highest pending IPL.

 

IPI Support

IRQ-REQ-030:

Support reservation-break IPI vector (0x510) at IPL 20.

IRQ-REQ-031:

Provide sendIPI(source, targetCPU, ipl, vector) for SMP coordination.

IRQ-REQ-032:

Broadcast IPIs must skip source CPU.
