### **Phase 4: Common Issues to Watch For**

 

#### **Issue 1: Grain Not Found (nullptr)**

```

SYMPTOM: slot.di.grain == nullptr after lookup

 

CHECK:

✓ What opcode was fetched?

✓ What function code was extracted?

✓ Is this grain registered?

✓ Run: InstructionGrainRegistry::instance().grainCount()

✓ Expected: > 0 (should see 100+ after all registrations)

✓ If 0: Registration phase failed!

 

REPORT TO ME:

"Grain lookup returned nullptr for opcode 0x??, function 0x???.

Registry has ?? grains registered.

Instruction bits: 0x????????"

```

 

#### **Issue 2: Wrong Grain Assigned**

```

SYMPTOM: grain->opcode() doesn't match di.opcode

 

CHECK:

✓ grain->opcode() returns: 0x??

✓ Expected opcode: 0x??

✓ grain->functionCode() returns: 0x???

✓ Expected function: 0x???

 

REPORT TO ME:

"Grain mismatch!

Expected: opcode 0x??, function 0x???

Got grain with: opcode 0x??, function 0x???

Grain type: [from debugger]"

```

 

#### **Issue 3: Instruction Decode Errors**

```

SYMPTOM: Opcode/function extracted wrong

 

CHECK:

✓ Raw bits: 0x????????

✓ Bit extraction for opcode (31:26):

 - Manual calc: (rawBits >> 26) & 0x3F = 0x??

 - di.opcode = 0x??

 - Match? [Yes/No]

 

REPORT TO ME:

"Instruction decode error:

Raw bits: 0x????????

Extracted opcode: 0x?? (should be 0x??)

Extracted function: 0x??? (should be 0x???)"

```

 

#### **Issue 4: Grain Registry Empty**

```

SYMPTOM: grainCount() returns 0

 

CHECK:

✓ Did Phase 2 initialization complete?

✓ Look for log: "Instruction grains registered: ??"

✓ Check if GrainRegistration.cpp was compiled/linked

 

REPORT TO ME:

"Grain registry is empty!

Phase 2 initialization status: [completed/failed]

GrainRegistration.cpp in build: [yes/no/unknown]"

```

### **Phase 5: Step-by-Step Debugging Script**

 

**Use this script when debugging:**

```

1. START DEBUGGER

  Set all breakpoints listed in Phase 1

 

2. RUN TO FIRST FETCH

  When hit:

  - Note PC value

  - Note memory address being fetched

  - Step forward, capture raw instruction bits

 

3. STEP INTO DECODE

  When hit:

  - Verify rawBits matches what fetch returned

  - Watch opcode extraction

  - Watch function extraction

  - Verify values are sane (opcode 0x00-0x3F)

 

4. STEP INTO GRAIN LOOKUP

  When hit:

  - Note lookup key calculation

  - Check registry size (should be 100+)

  - Watch grain pointer returned

  - If nullptr, STOP and report

 

5. STEP INTO SLOT ASSIGNMENT

  When hit:

  - Verify grain pointer copied correctly

  - Verify di.grain is not nullptr

  - Verify slot is marked valid

 

6. REPORT STATUS

  Copy the captured values using templates above

```
